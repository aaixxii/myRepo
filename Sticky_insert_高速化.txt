1. SYSTEM_CONFIG テーブルに、PARALLEL_SEGMENTS_COUNT、PARALLEL_SEGMENTS_ALGORITHM（RoundRobin,Sticky）、STICKY_COUNTとの三つ項目を追加
2. MM DBにPARALLEL_SEGMENTS_INFOテーブルを追加
    CREATE TABLE PARALLEL_SEGMENTS_INFO (
      ID                    INT(8)  NOT NULL,
      SEGMENT_ID             BIGINT(38),
      USED_STICKY_COUNT　　　　　INT(8)
      LAST_USING_SEGMENT     BOOLEN,
      PRIMARY KEY(ID)
    ) ENGINE=INNODB;
3. add_biometrics プロシージャ IN パラメータに、init_semgnet(boolean)とupdate_segment_id(bigint)を追加
4. javaでSyncInsert最初に、まず以下のようにどのSegmentを使うを確認(decide_segment プロシージャ新規)：
4.1 SYSTEM_CONFIG テーブルから、PARALLEL_SEGMENTS_COUNT、PARALLEL_SEGMENTS_ALGORITHM、STICKY_COUNTを取得
4.2 PARALLEL_SEGMENTS_INFOテーブルから、COUNT(SEGMENT_ID)を取得
4.2.1 COUNT(SEGMENT_ID)< PARALLEL_SEGMENTS_COUNTの場合、
　　　　　init_semgnet=true, update_segment_id=-999で、add_biometricsをCall（他のパラメータはいまのとおり）
4.2.2　COUNT(SEGMENT_ID)>= PARALLEL_SEGMENTS_COUNTの場合、
　　　　　　 PARALLEL_SEGMENTS_INFOテーブル情報より、どうのSegmentIDを使うを決める：　　　　　 　
　　　　　　　  PARALLEL_SEGMENTS_INFO.LAST_USING_SEGMENTが全部falseの場合(最初)、
　　　　　　　　　　 MinSegmentIdを選択、このレコード中のLAST_USING_SEGMENTを１(true)に設定
             init_semgnet=false, update_segment_id=Min SEGMENT_IDで、add_biometricsをCall（他のパラメータはいまのとおり）
　　　　　　　　PARALLEL_SEGMENTS_INFO.LAST_USING_SEGMENTが設定されて、、かつPARALLEL_SEGMENTS_ALGORITHM=Stickyの場合、          
　　　　　　　　  PARALLEL_SEGMENTS_INFOより、LAST_USING_SEGMENT=1(true)のレコードを選択、
            上記レコード.USED_STICKY_COUNTを確認：
	  　　  LAST_USING_SEGMENT.USED_STICKY_COUNT　<=　SYSTEM_CONFIG.STICKY_COUNTの場合、
	  　　　　  init_semgnet=false, update_segment_id=該当レコードのSEGMENT_IDで、add_biometricsをCall（他のパラメータはいまのとおり）
	  　　  LAST_USING_SEGMENT.USED_STICKY_COUNT　>　SYSTEM_CONFIG.STICKY_COUNTの場合、
	  　　　　 round robinで次のSegmentIdを選択し、このレコード中のLAST_USING_SEGMENTを１(true)に設定、 同時に前にLAST_USING_SEGMENT欄の１(true)を0(false)に変更　
                init_semgnet=false、上記決めたSEGMENT_IDで、add_biometricsをCall（他のパラメータはいまのとおり）	  　　　　　　　　　　 
5．add_biometrics プロシージャ内容の修正：   
5.1 MAX(SEGMENT_ID)を更新ではなく、IN　パラメータに伝われたupdate_segment_idを更新する。
5.2　SEGMENTS更新場合、
5.2.1　UPDATE SEGMENTS条件に、init_semgnetを追加：
　　　　　IF init_semgnet==false AND (seg_record_count  < tmp_max_seg_record_count) 
5.2.2 PARALLEL_SEGMENTS_INFOより、USED_STICKY_COUNTを更新　
　　　　　USED_STICKY_COUNT >= SYSTEM_CONFIG.STICKY_COUNTの場合、USED_STICKY_COUNT＝1に設定
　　　　　USED_STICKY_COUNT < SYSTEM_CONFIG.STICKY_COUNTの場合USED_STICKY_COUNT＝USED_STICKY_COUNT+1に更新
5.3　Segment更新後、max_seg_record_countに達しているかを確認
　　　　　max_seg_record_countに到達した場合、PARALLEL_SEGMENTS_INFOテーブルから、該当SegmentIDのレコードを削除　　　　　　
5.4　NewSegmentの場合、
　　　PARALLEL_SEGMENTS_INFOテーブルに、該当SegmentIDを追加、該当レコードのLAST_USING_SEGMENT=0(false)に設定、該当レコードのUSED_STICKY_COUNT=1に設定。
